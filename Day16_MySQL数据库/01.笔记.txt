数据库 - 数据的仓库（集散地）--database - 实现数据持久化和数据管理持久化 - 将数据从内存转移到能够长久保存数据的存储介质的过程数据库的分类：关系型数据库（SQL）和非关系型数据库（NoSQL）文件系统 / 层次数据库/ 网状数据库关系型数据库- 1970s - E.F.Codd - IBM研究员 - System R- 理论基础：关系代数和集合论- 具体表象：用二维表来保存数据 - 学生表    ~ 行：一条记录 - 一个学生的信息    ~ 列：一个字段 - 学生的某个属性， 如：学号，姓名，出生日期    ~ 主键列： 能够唯一标识一条记录的列，如：学号- 编程语言：SQL - 结构化查询语言    ~ DDL - 数据定义语言 - create / drop / alter    ~ DML - 数据操作语言 - insert / delete / update / select (***)    ~ DCL - 数据控制语言 - grant（授权） / revoke（召回）LAMP = Linux + Apache + MySQL + PHPphp -> javaMySQL -> OracleLinux -> 小型机关系型数据库产品：- Oracle - 甲骨文- IBM DB2- Microsoft SQLServer- Sybase---------------------- MySQL- PostgreSQL- SQLiteExample:学生（学号，姓名，性别，生日，家庭地址）学院（编号，名称，院长，网址，...）教师（工号，姓名，职称，所属学院）课程表（编号，名称，学分，授课老师）选课记录表（选课记录编号，选课学生，所选课程，选课时间，考试成绩）插入不少于3条记录实体：学生、学院关系：属于 （学生属于学院）重数：N:1 多对一注意事项：    1.给数据库和表命名的时候尽量使用小写；    2.作为筛选条件的字符串是否区分大小写看设置的校对规则    3.数据库中的对象通常会用前缀加以区分        table/view/index/function/procedure/trigger
